## 2. Padrão dos erros

Seus erros mais frequentes foram estes:

**1. Confusão entre passado simples e “be” + particípio**
Você escreveu várias vezes:

* *They’re validated*
* *She’s reviewed*
* *We are finished*

Mas o correto muitas vezes era:

* **They validated**
* **She reviewed**
* **We finished / We have finished**

**2. Verbos irregulares**
Você errou especialmente:

* *finded* → **found**
* *writed* → **wrote / written**
  No seu exercício, o mais natural era muitas vezes trocar o verbo inteiro:
* **documented** em vez de *writed*

**3. Uso de pronome para coisas**
Para sistema, servidor, API, branch, task, plugin:

* use **it**
* não use **he/she**

**4. Vocabulário pouco natural**

* *flux* → **flow**
* *homolog* → **staging environment**
* *deploy configurations* → **deployment settings**
* *infra* → **infrastructure team**

**5. Ortografia**

* *bussiness* → **business**
* *analising* → **analysing/analyzing**
* *indetified* → **identified**
* *reviewd* → **reviewed**
* *aproved* → **approved**
* *avaliable* → **available**

## 3. Regra para memorizar

Memorize estas 4 regras:

**Regra 1 — passado simples**

* I validated
* You validated
* He validated
* They validated

Não use:

* *They’re validated*
  quando você quer dizer “eles validaram”.

**Regra 2 — com “did”, o verbo volta ao normal**

* **Did you check...?**
* **Did you find...?**

Não:

* *Did you checked*
* *Did you found*

**Regra 3 — coisas usam “it”**

* The system is slow, but **it** is working.
* The API is unstable, but **it** still responds.
* The branch is ready, and **it** can be reviewed.

**Regra 4 — verbo irregular importante**

* find → found
* write → wrote / written
